## 유니티의 라이프 사이클

유니티에는 라이프 사이클이 존재한다.
게임이 시작되고, 끝날 때까지 특정 주기마다 호출되는 기본 메서드들이 있다.

`게임 실행`  →  `Awake()`  →  `OnEnable()`  →  `Start()`  →  `Update()`  →   `FixedUpdate()`  →  `LateUpdate()`  →  `OnDisable()`  →  `OnDestroy()`

유니티에서는 이 순서대로 게임이 실행된다

-----------------

### **1. 초기화 구간 (시작 단계)**
- **Awake()**
	- 객체가 생성되자마자 한 번만 실행되며 가장 먼저 호출된다.
	- 객체의 활성, 비활성화 상관없이 실행된다.
	- 보통 싱글톤이나 컴포넌트 참조를 위해 사용한다.


		(컴포넌트 참조: rb = GetComponent<Rigidbody>();)
<br/>

- **OnEnable()**
	- 객체가 활성화될 때마다 호출된다.
	- 이벤트 등록이나 코루틴에 사용된다.
<br/>

- **Start()**
	- 첫 Update() 함수가 호출되기 전, 딱 한 번 실행된다.
	- 주로 초기 UI 세팅 등에 사용된다.
<br/>

-----------------

### **2. 게임 진행**
- **Update()**
	- 매 프레임마다 호출된다.
	- 주로 키 입력, 캐릭터 이동, 타이머 감소에 사용된다.
	* 매 프레임마다 호출된다는 것은 컴퓨터 성능에 따라 호출 횟수가 달라진다는 것이다.
<br/>

- **FixedUpdate()**
	- 프레임 간격과 관계없이 일정한 시간마다 호출된다.
	- Rigidbody 이동, 물리 연산 등에 사용된다.
<br/>

- **LateUpdate()**
	- Update()가 끝난 뒤 호출된다.
	- 카메라 따라가기나 Update 결과를 반영하는 데 주로 사용된다.
<br/>


-----------------

### **3. 비활성화, 종료**
- **OnDisable()**
	- 객체가 비활성화 될 때 호출된다.
	- 주로 이벤트 해제나 코루틴을 정지시킬 때 사용한다.
<br/>

- **Destroy()**
	- 객체가 파괴될 때 호출된다.
	- 메모리 정리나 데이터 저장 등에 사용된다.
<br/>

-----------------

만약 Update() 중간에 씬 전환을 할 경우 그 프레임의 Update()가 끝까지 돌고나서 모든 객체들이 OnDisable(), Destroy()를 호출하고 새로운 씬으로 전환된다.

(씬 전환: Scenemanager.LoadScene("씬 이름");)
