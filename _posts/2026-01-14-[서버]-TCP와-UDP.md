## TCP와 UDP

분명 정보처리기사 시험 때 공부했는 왜 가물가물하지....

**TCP**와 **UDP**는 인터넷에서 데이터를 주고받을 때 사용하는 대표적인 전송 계층의 프로토콜이다.
어떻게 데이터를 주고 받을 것인가에 대한 약속이다.

### **1. TCP (Transmission Control Protocol)**
#### **TCP의 특징**
1. **연결 기반** : 통신 전에 3-way handshake를 통해 연결을 설정한다.
2. **신뢰성 보장** : 패킷 손실 시 재전송과 순서를 보장한다.
3. **흐름 제어**와 **혼합 제어**를 지원한다.
4. **에러 체크**와 복구가 가능하다.

#### **TCP의 장단점**
1. **장점**
	- 데이터 정확성을 보장한다.
	- 보낸 순서를 유지한다.
	- 통신 중에 에러 복구가 가능하다.

2. **단점**
	- 속도가 느리다.
	- 실시간 통신에 부적합하다.

##### **3-way handshake**
**3-way handshake**란 클라이언트와 서버가 서로 신호를 주고 받는 3단계 절차이다.
- 클라이언트 → 서버 : 연결 요청
- 서버 → 클라이언트 : 요청 수락 및 응답
- 클라이언트 → 서버 : 응답 확인

##### **예시**
- 웹 브라우징 (HTTP/HTTPS)
-   이메일 (SMTP, IMAP, POP3)
-   파일 전송 (FTP)

### **2. UDP (User Datagram Protocol)**
#### **UDP의 특징**
1. **비연결 기반** : 연결없이 바로 데이터를 전송한다.
2. **신뢰성 비보장** : 순서가 보장되지 않고, 손실 시 재전송을 하지 않는다.
3. 오버헤드가 적고 빠른 전송이 가능하다.
4. 헤더 구조가 단순하다.

#### **절차지향 언어의 장단점**
1. **장점**
	- 전송 속도가 빠르다.
	- 실시간 전송에 유리하다.

2. **단점**
	- 데이터가 손실될 가능성이 있다.
	- 순서가 보장되지 않는다.
	- 에러를 복구할 수 없다.

##### **예시**
-   실시간 스트리밍 (YouTube Live, Zoom)
-   온라인 게임
-   음성 통화 (VoIP, 예: 카카오톡 음성 통화)
