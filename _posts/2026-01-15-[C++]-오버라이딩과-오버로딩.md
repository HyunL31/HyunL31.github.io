## 오버라이딩과 오버로딩

### **1. 오버라이딩 (Overriding)**
**오버라이딩**이란 메서드를 재정의하는 것을 의미한다.
부모 클래스의 메서드를 자식 클래서에서 재정의한다.

#### **오버라이딩의 특징**
1. 부모 클래스와 자식 클래스 사이에서의 메서드 재정의이기에 상속이 반드시 필요하다.
2. 메서드의 이름, 반환 타입, 매개변수가 똑같아야 한다.
3. 다형성과 관련이 있다.

##### **예시**
```cpp
// 부모 클래스
class Parent
{
public:
    virtual void func()
    {
        cout << "Parent\n";
    }
};

// 자식 클래스
class Child : public Parent
{
public:
    void func() override	// 이름, 반환 타입, 매개변수 동일
    {
        cout << "Child\n";
    }
};

int main()
{
	Parent* p = new Child();
	p->func();   // Child 출력
	
	return 0;
}
```

### **2. 오버로딩 (Overloading)**
**오버로딩**이란 매개변수만 다른 메서드를 여러 개 정의하는 것을 의미한다.

#### **오버로딩의 특징**
1. 오버라이딩과 달리 상속이 필요하지 않다.
2. 메서드 이름은 같고 매개변수 개수나 타입이 달라야 한다.
3. 반환 타입만 다른 것은 오버로딩이 불가능하다.
4. 헤더 구조가 단순하다.

##### **예시**
```cpp
class Calculator {
public:
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
};
```
